Create or Open a Contest

Click "File"->"New or Open" in the main menu, or hit "Ctrl+O" to open the "New or Open" dialog where you can create new tests or open an existing one. The dialog may seem like this:

You can create a new test in the "New" page, or open an existing one in the "Recent" and "Specified" pages.

New

In this page, you can create new tests.

The meanings are listed as follows:

Contest Title
The title for the new contest. It is "test on $(date)" by default, where $(date) is the current date string in the format "HHHH-MM-DD".
NOTE: this should not be left blank.
Save In
The path of the folder where the files will be stored. It is "$(folder)$(title)" by default, where $(folder) is the "Default Test Directory" in the "Options" dialog, and $(title) is the text filled in the "Contest Title" field.
NOTE: this should not be left blank.

Recent

This is the default page. In this page, you can open a contest you recently closed.

The meanings are listed as follows:

Contests
A list of the contests you recently closed. When an item is highlighted, its information will be shown in the right part.
You can reload the list by clicking "Refresh" in the pop-up menu. You can delete an entry by highlighting it and clicking "Delete" in the pop-up menu.
Folder
The path of the folder where the files for this test are stored.
Problems
An overview of the problems in this contest.
The information is given in the format "$(title)($(source));", where ${title} is the problem title, and $(source) is the text filled in the "Source File" field of the data configuration.

Specified

In this page, you can open a contest by selecting the folder where it is stored.

Highlight the folder in the tree view, and its absolute path is shown in the "Directory" field.
NOTE: the default test directory will be highlighted at first.


Click the OK button to implement the action, or click the Cancel button to cancel.

Configure Data

SILICA perform the judge task according to the data configuration of the contest.
A contest consists of several problems, and a problem consists of several testcases, as is shown in the tree view. In the configuration procedure, you should configure both problems and testcases.

The Dropdown List

SILICA provides a dropdown list, which simplifies the configuration efficiently.

The dropdown list will pop up when you click one of those input boxes, and the items in the list will be sorted descending in correlation with the input.

Configure Problems

You can click Add a Problem in the pop-up menu, or hit Ctrl+Ins to add a problem. You can click Delete in the pop-up menu, or hit Del to delete the highlighted problem.

When a problem entry in the tree view is selected, its configuration is shown in the Detail panel.

The meanings are listed as follows:

Problem Title
The title of the problem.
NOTE: this should not be left blank.
Source File
The filename of the submissions, without extension.
In the compilation procedure, SILICA will try to find the submission $(source)$(ext), where $(source) is the text filled in this section, and $(ext) is the extension (with leading dot) of a compiler.
NOTE: this should not be left blank.
Input File
The name of the input file for the problem. This is usually given in the problem description.
Output File
The name of the output file for the problem. This is usually given in the problem description.
Compare Method
How to compare the competitor's outputs with the standard ones. There are three methods available:
byte by byte
Two files are considered identical if all bytes are exactly the same.
ignore spare whitespaces
Whitespaces (space, tab and CR/LF) are out of consideration when comparing.
use custom checker
User's output will be judged and scored by an external checker.
Custom Checker
The relative path of the custom checker. This is enabled only if Compare Method is set to use custom checker.
NOTE: make sure you have the permission to run the checker.
Additional File
The relative path of the additional file for this problem. Leave blank if none.

Configure Testcases

You can click Add a Testcase in the pop-up menu, or hit Ins to add a testcase to the highlighted problem. You can click Delete in the pop-up menu, or hit Del to delete the highlighted testcase. You can click Delete Other Testcases in the pop-up menu, or hit Shift+ Del to delete testcases other than the highlighted one.

When a testcase entry in the tree view is selected, its configuration is shown in the Detail panel.

The meanings are listed as follows:

Input File
The relative path of the standard input file for this testcase. You can choose one in the dropdown list.
Output File
The relative path of the standard output file for this testcase. You can choose one in the dropdown list.
Full Score
This integer is the full score for this testcase.
Time Limit
This integer is the time limit for this testcase (in seconds).
Memory Limit
This integer is the memory limit for this testcase (in kB).

HINT: after creating the first testcase, you can click Add Other Testcases in the pop-up menu, or hit Shift+Ins to add more testcases. SILICA will try to analyse the paths of the input and output files of the first testcase, try to find more files of the same pattern and create testcases using them.

Judge Contestants

Each contestant should create a folder named after his name under the subdirectory "src" relative to the contest directory, in which his submissions are stored.
In the "Contestants" page, a list shows all the competitors and related information such as rank and score.

You can click a column header to sort contestants by that column. You can click Rename in the pop-up menu, or hit F2 to change the name of the highlighted competitor. You can click Delete in the pop-up menu, or hit delete to delete all the submissions of the highlighted competitor. You can click View Source in the pop-up menu to open the directory where the highlighted competitor's sources are located, using the file manager.

Having configured the data, you can click the Test All button to judge all the contestants, or click Test Selected button to judge those highlighted.

The test form will pop up when the judge task is started, in which the status is updated dynamically.

You can click the Pause button to suspend judging. You can click the Abort button to terminate judging.
WARNING: it is very dangerous to terminate the judging thread, and may lead to unexpected consequences.

View Contestant's Result

You can click View Result in the pop-up menu, or perform a double-click to see the test result of the highlighted competitor.

The list on the left side provides an overview of the results. When an item is highlighted, its detail will be shown in the list on the right side.

You can click the Rejudge Selected button to judge the checked problems. You can click the Rejudge All button to judge all the problems.

Export Contest Result

SILICA is able to export the result of the current contest to text files.

To make a report, click "Control"->"Export Result" or hit "Ctrl+E" to open the "Export" dialog, which may seem like this:

You can edit the absolute path of the report in the "Save As" field. Click the Export button to export the results of highlighted contestants, or click the Cancel button to abort.
The results can be exported in two ways. Simple and Detailed.

Simple
Each line demonstrates an overview of the scores a contestant achieved.
Detailed
Each line demonstrates a detailed description of the performance of a contestant. The meanings of the symbols are listed at the end of the report.

Package a Contest

SILICA is able to package the contest into a tarball.

Check "With sources & results" to package the "src" folder. Check "With dataconf.xml" to package the data configuration.
Click OK to package the contest, or click Cancel to abort.

Close a Contest

Click "File"->"Close" in the main menu, or hit "Ctrl+W" to close the contest.
