About SILICA

Olympiad in Informatics(OI) is a contest for high school students. There are a lot of OI judgers for M$ Window$, while there are few for GNU/Linux. Hence there comes SILICA, a software designed to help people who are devoted to Olympiad in Informatics(OI) using GNU/Linux.

SILICA 0.3 is the third and latest release in sequence. This release consists of two components:
silica
The main component of SIILCA 0.3, serves as the graphical frontend for silica-judge. The interface is inspired by cena, and is easy for users to get started.
silica is developed using Lazarus, and is compiled with the GTK+ 2 widgets by default.
silica-judge
The core utility of SILICA 0.3, undertaking the most important tasks: run & judge. It is inspired by rjudge.
silica-judge is developed using C++.

Why SILICA

SILICA is short for "SImple LIghtweight Contest Assistant". As its name entails, SILICA aims at being a simple-but-competent OI judger.

SILICA works under GNU/Linux, at least Ubuntu. It is brought to you with the following features:

cena-like GUI
Cena might be the most popular OI judger for M$ Window$. SILICA's cena-like GUI makes it easier for users to switch from cena to SILICA.
open source
SILICA is an open source software, which means you can download the source and compile it, modify it, and learn from it. You can get a copy of the source codes from our website.
custom compilers
You can add other compilers besides fpc,gcc, g++and bash scripts, which are the default compilers.
package current contest
SILICA is able to package the files of the contest into a tarball(uncompressed), so that you can back it up or share it with others.
simplified data configuration
SILICA provides a dropdown list which collects all the files related to the problem, sorted descending in correlation. You can finish the configuration by just typing a few letter and then click the mouse.
export contest results
SILICA is able to export the results of the contest in two ways: detailed and simple.
inspect memory usage
SILICA can do more than restricting memory allocation for a program. It can collect infomation about the program's approximate memory usage.
NOTE: the numbers represents the maximum memory usage given by the linux kernel.
flexible judging queue
SILICA 0.3 makes it possible to judge some selected contestant in the "Contestants" page. It is also feasible to re-judge selected problems for a certain competitor.
