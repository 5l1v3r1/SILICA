Options

Before you start the first contest, you had better check the options, since the options generated by SILICA may not be compatable with your operating system.

To view the options, click "Options"->"Options" in the main menu or hit "Ctrl+P" to open the Options Dialog, which may seem like this:

The meanings are listed as follows:

Default Memory Limit
This integer is the default memory limit (in kB) for a testcase when doing the data configuration.
It is 65536 by default.
Default Time Limit
This integer is the default time limit (in seconds) for a testcase when doing the data configuration.
It is 1 by default.
Default Score
This integer is the default full score for a testcase when doing the data configuration.
It is 10 by default.
Default Test Directory
This should be directory where most of your contests are stored. When starting a new contest, it will be saved in this folder by default.
It is your home directory by default.
Directory Explorer
The command to launch a graphical file manager. SILICA can use the file manager to open the contest folder. Some examples of file managers are Nautilus for gnome and Konqurer for KDE.
It is /usr/bin/nautilus by default.
Web Explorer
The command to launch a web browser. SILICA can use the browser to open this document. Some examples of web browsers are Firefox, Konqurer, Chromium and Opera.
It is /usr/bin/firefox by default.

Click the OK button to apply the changes and close the form. Click the Cancel button to discard the changes and close the form.

Compilers

When compiling the source code submitted, SILICA will try the registered compilers in sequence.
SILICA offers four compilers by default: fpc, gcc, g++ and bash scripts. You can add some custom compilers.

To open the "Compilers" dialog, click "Options"->"Compilers" in the main menu. The dialog may seem like this:

You can register a new compiler by clicking the New button. You can erase the highlighted registration by clicking the Delete button. You can change the order of the highlighted compiler by clicking the Move Up and Move Down button.

NOTE: SILICA will use the first matching compiler in the sequence shown in the list.

When one item in the list is highlighted, the detailed information will be shown in the "Detail" panel. The meanings are listed as follows:

Name
The name of this compiler.
NOTE: this should not be left blank.
Extension
The extension of the source files to be compiled using this compiler.
NOTE: the extension can be blank; if not, it should start with a dot (".",without quotation).
Script Language
This checkbox is checked if this is the compiler for a script language, such as bash.
NOTE: sources in script languages ususally need not compilation.
Compilation Command
The execution command for the compilation.
eg: fpc %s -o%s -XS
NOTE: The first %s will be replaced by $(source)$(extension), and the second %s will be replaced by $(source), where $(source) is the text in the "Source File" field of the data configuration, and $(extension) is the text in the "Extension" filed of the compiler.
Execution Command
The execution command to launch the the program just compiled.
eg: ./%s
NOTE: The %s will be replaced by $(source), where $(source) is the text filled in the "Source File" section of the data configuration.

Click the OK button to apply the changes and close the form. Click the Cancel button to discard the changes and close the form.
